## 🔍 詳細

### ⌚ 任意スラッシュコマンド

- `/watch <metric>` … 使用率/温度を `BarChart`・ゲージでライブ監視（一過性 UI のため高さ限定で `Live`/`Status`）。
- `/copy` `/save` … 直近のエージェント応答をクリップボード/ファイルへ。

### 🧪 内容面の評価

- **数値妥当性**: 応答中の値を、ツールが返したスナップショット（実測値）や妥当レンジと自前ロジックで照合する。
- **診断的確さ**: 決定的なルールエンジンの `Findings` を正解として、LLM の指摘・要約と照合する（正解が既知のため判定に LLM 不要）。
- 現状の評価（`PcAgent.Evaluation` / `LocalEvaluator`）は**振る舞い検査**に留める。詳細は [spec §6.2](spec.md#62-任意機能の判断)。

### 📡 収集源の追加

- `ICollector` を実装し DI 登録するだけ（**Wi-Fi コレクタ（ManagedNativeWifi）と同じ要領**で実装済み実績あり）。イベントログ、追加の温度センサー等。

---

## 🚫 不採用（方針）

- **LLM-as-judge 評価**（`Microsoft.Extensions.AI.Evaluation`）… ユーザー判断により不採用。
- **マルチエージェント**（ワークフロー / A2A）… 単一ドメインのため。
- **構造化出力**（`RunAsync<T>`）… 診断レポートはルールが直接生成するため補助のみ。
